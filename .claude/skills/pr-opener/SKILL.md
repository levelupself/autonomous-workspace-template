# PR-Opener Skill

Open a pull request for the completed stage.

## Step 1 — Verify preconditions

```bash
# Confirm we're on an agent worktree branch (not main)
git branch --show-current

# Confirm there are commits ahead of main
git log main..HEAD --oneline

# Confirm tests pass (should already be done — double check)
```

If on main or no commits ahead, stop with an error.

## Step 2 — Collect PR content

Read:
- Current stage name and number from `docs/stages.md`
- Completed task list from `docs/stages.md`
- Any decisions logged in `docs/decisions.md` this session
- Commit messages: `git log main..HEAD --format="%s"`

## Step 3 — Generate PR body

```markdown
## Summary

Stage <n>: <stage name> for <project name>.

## Changes

<list of completed tasks from stages.md — bulleted>

## Test coverage

<describe tests added and their results>

## Decisions made

<any architectural decisions from this session — brief>

## Tokens used

<total for this session>

## Next stage

<name of next stage and what it will cover>

---
_Generated by autonomous agent. Review diff before merging._
```

## Step 4 — Open PR

```bash
gh pr create \
  --title "<project>: stage <n> — <stage name>" \
  --body "<generated body>" \
  --base main \
  --head "$(git branch --show-current)" \
  --label "agent-generated"
```

If `gh` is not available, write the PR details to
`docs/pr-draft-stage-<n>.md` and note that it needs to be opened manually.

## Step 5 — Write PR URL to stages.md

```bash
PR_URL=$(gh pr view --json url -q .url 2>/dev/null || echo "pending")
```

Append to the completed stage in stages.md:
```
**pr:** <URL>
```
